### ## Part 1: 상표 유형 선택 UI/UX 최적화

**목표**: 현재의 상표 분석 시작 UI를 사용자가 각 상표 유형을 쉽게 이해하고 선택할 수 있도록, 예시가 포함된 명확한 선택 화면으로 교체합니다.

**요구사항:**

1.  기존 `analysis-page-content.tsx`의 상단 부분을 수정하여, 사용자에게 가장 먼저 세 가지 상표 유형 선택지를 보여주세요.
2.  각 선택지는 **카드(Card) 형태의 컴포넌트**를 사용하여 시각적으로 구분해주세요.
3.  각 카드에는 **유형 이름, 명확한 설명, 그리고 구체적인 예시**가 포함되어야 합니다. 아래 텍스트를 정확히 사용해주세요.

    * **카드 1: 문자 상표**
        * **설명:** 한글, 영어, 한자 등 문자만으로 구성된 상표
        * **예시:** "삼성", "SAMSUNG"

    * **카드 2: 도형 상표**
        * **설명:** 그림, 로고, 아이콘 등 도형만으로 구성된 상표
        * **예시:** 나이키의 스우시 로고 이미지 (아이콘 사용 가능)

    * **카드 3: 결합 상표**
        * **설명:** 문자와 도형이 함께 결합된 형태로, 가장 흔하게 볼 수 있는 유형입니다.
        * **예시:** 스타벅스 로고와 "STARBUCKS" 문자가 결합된 형태 (아이콘과 텍스트 조합)

4.  사용자가 세 카드 중 하나를 클릭하면, 해당 선택이 활성화되는 시각적 피드백(예: 테두리 색상 변경)을 주세요.
5.  선택 후 하단에 나타나는 입력 폼은 선택된 유형에 맞게 최적화되어야 합니다. (예: '도형 상표' 선택 시 이미지 업로드 강조)
6.  **CSS 스타일:** 기존 UI(`image_0c1323.png` 참고)의 **색상 팔레트, 버튼 스타일, 라운드 값 등 디자인 요소를 그대로 유지**하여 일관된 사용자 경험을 제공해야 합니다.

### ## Part 2: LangGraph 기반 대화형 분석 UI/UX 신규 개발

**목표**: Part 1에서 초기 정보 입력을 완료한 후, 기존의 복잡한 분석 UI를 모두 제거하고, LangGraph 기반의 전문가와 대화하는 듯한 새로운 채팅 UI로 완전히 교체합니다.

**요구사항:**

1.  초기 정보 제출 후, 화면은 **채팅 UI**로 전환됩니다.
2.  이 채팅 UI는 LangGraph 백엔드 API(`/api/analysis/chat`)와 통신하며, API 응답의 `status` 값 (`QUESTION`, `ANALYZING`, `CONFIRMATION_REQUIRED`, `COMPLETE`)에 따라 동적으로 변화해야 합니다.
3.  **각 상태(Status)별 UI 구현:**
    * **`QUESTION` 상태:**
        * AI가 보낸 질문이 채팅 버블로 나타납니다.
        * 하단에는 사용자가 답변을 입력할 수 있는 텍스트 입력 필드와 [전송] 버튼이 있어야 합니다.
    * **`ANALYZING` 상태:**
        * "감사합니다. 분석을 시작하겠습니다..."와 같은 메시지를 보여줍니다.
        * 입력창을 비활성화하고, 분석이 진행 중임을 알리는 로딩 인디케이터나 프로그레스 바를 표시합니다.
    * **`CONFIRMATION_REQUIRED` 상태 (중요):**
        * AI가 분석 중 발견한 이슈(예: "유사한 상표 'PAX KOREA'를 발견했습니다.")를 설명하고, 사용자에게 확인을 요청하는 메시지를 채팅 버블로 보여줍니다.
        * 사용자가 답변할 수 있도록 입력창을 다시 활성화합니다.
    * **`COMPLETE` 상태:**
        * "분석이 완료되었습니다." 메시지와 함께, 최종 보고서 페이지로 이동할 수 있는 [결과 확인하기] 버튼을 표시합니다.

4.  **CSS 스타일:** 전체적인 디자인은 Part 1 및 기존 UI의 테마와 완벽하게 일관성을 유지해야 합니다. Shadcn UI의 `Card`, `Input`, `Button` 컴포넌트를 적극 활용하여 통일성을 지켜주세요.
5.  **구현 파일:** 이 새로운 대화형 분석 UI는 기존 `analysis-page-content.tsx`의 하단 부분을 대체하거나, 새로운 컴포넌트(예: `conversational-analysis.tsx`)로 분리하여 구현해주세요.
